Mechanical & Actuation Build

Final Integrated Six-Leg Assembly

The final robot has all six legs mounted to the central frame with onboard wiring routed through the body. This is the physical configuration shown in the April demo media.

Single-Leg Prototype

The team built and iterated on a physical leg prototype to validate link dimensions, mounting clearances, and bracket geometry before full chassis assembly.

Servo Bracket Revision

V2 bracket revisions improved the fit for the Spektrum S6510 servos and aligned the mounting pattern more reliably with the intended mechanical stack.

Bench-Tested Single-Leg Motion

A hardware test leg now executes controller-generated motion on the bench. This validates the software-to-servo pipeline: IK targets are converted into PWM commands and passed through the embedded stack to the physical actuator.

Electronics & Embedded Control

Power & Wiring Bench

The team assembled a dedicated wiring bench for servo power distribution, PWM breakout hardware, and early integration checks. This setup supports incremental hardware validation before everything is routed into the final robot body.

ESP32 Control Layer

The ESP32 serves as the hardware-side control layer for command handling and PWM output. Its bench setup was used to bring up communication, servo direction settings, and actuator command routing.

Software Stack

Inverse Kinematics & Gait Control

The Python control stack computes coxa, femur, and tibia angles from body-relative foot targets, then maps those targets to hardware-ready servo commands. The same motion logic feeds simulation, hardware control, calibration, and dashboard workflows.

Interactive IK Demo

Before touching hardware, the team validated the stepping trajectory, servo angle mapping, and body-frame transforms using a single-leg visualization environment.

Controller-Oriented Motion Demo

A full-body controller demo validates the command path from high-level drive inputs to synchronized multi-leg gait output.

Simulation & Vision Integration

MuJoCo Physics Validation

The team replaced a simple matplotlib viewer with a MuJoCo environment that models gravity, contact, joint limits, and actuator commands. The simulation assets share leg geometry and gait assumptions with the Python IK stack.

Raspberry Pi Camera Bring-Up

The Raspberry Pi 5 camera stack was brought up first for streaming, then extended into an AprilTag detection pipeline for calibration and future pose-aware behaviors.

AprilTag Detection Pipeline

The Pi detects multiple AprilTags using OpenCV's AprilTag dictionary. The implementation provides a concrete path toward calibration, localization, and pose-aware navigation.

Operator Dashboard

The Raspberry Pi now hosts a user-friendly browser dashboard at the robot control endpoint. The interface wraps the runtime, manual driving, servo calibration, camera console, automated mode, and dance recording tools so the robot can be operated without command-line access during demos.
